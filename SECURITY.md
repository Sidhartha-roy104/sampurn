# 🔐 Security Policy

## Supported Versions

Sampūrṇ is currently under active development.

| Version | Supported |
| ------- | ---------- |
| Latest Version | ✅ |
| Older Versions | ❌ |

---

# 🛡️ Reporting a Vulnerability

If you discover a security vulnerability in Sampūrṇ, please report it responsibly.

You can report vulnerabilities by:
- Opening a private security report on GitHub
- Or contacting the developer directly

Please include:
- Description of the issue
- Steps to reproduce
- Screenshots or proof-of-concept (if possible)
- Impact of the vulnerability

---

# ⚠️ Important Notes

Please do NOT:
- publicly disclose vulnerabilities before they are reviewed
- spam or abuse the application
- attempt to access other users' data

Responsible disclosure is appreciated.

---

# 🔒 Security Measures Used

Sampūrṇ currently uses:
- Firebase Authentication
- Firestore Security Rules
- Restricted Firebase API Key Domains
- Local browser storage for Pocket Notes

---

# ⏱️ Response Time

Security reports are usually reviewed within:
- 48–72 hours

Valid vulnerabilities will be investigated and fixed as soon as possible.

---

# 📌 Scope

This security policy applies to:
- the web application
- Firebase backend configuration
- authentication flows
- Pocket Notes storage system

---

Thank you for helping keep Sampūrṇ secure 💛
